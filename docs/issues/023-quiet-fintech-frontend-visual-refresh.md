# Quiet Fintech Frontend Visual Refresh

Labels: `needs-triage`

## Parent

`docs/prd.md`

## What to build

Refresh the frontend visual system from the current bright teal/green decorative presentation into a quieter fintech dashboard style. The product should feel like a serious wallet/security workflow: neutral surfaces, restrained primary color, readable status colors, compact information density, and no decorative background blobs or oversized marketing hero treatment.

## Acceptance criteria

- [ ] The homepage first viewport is operational, not a large marketing hero.
- [ ] Decorative radial blobs/orbs and high-saturation background effects are removed.
- [ ] The palette uses neutral background and surface colors with a restrained primary action color.
- [ ] Success, blocked, warning, and error colors are reserved for state communication and remain readable.
- [ ] Cards and panels use consistent compact radii, borders, spacing, and typography suitable for a dashboard.
- [ ] Text does not overflow buttons, tabs, cards, status strips, or activity log entries on mobile and desktop widths.
- [ ] The visual refresh preserves Indonesian and English copy support in the key demo flow.
- [ ] Frontend tests or screenshot checks cover the first viewport and the main DCA workflow at mobile and desktop widths.
