# Polet AI — Progress

## Status Summary

Polet AI is being redirected from a public policy smart wallet into a single-contract confidential DCA smart wallet for AI agents.

The old v1 work remains useful as prior art, but the final hackathon path may substantially refactor the contract, proxy, SDK, and frontend. The key privacy requirement is that plaintext numeric policy rules must not remain in the final confidential execution path.

Primary source of truth:

- PRD: `docs/prd.md`
- Local issues: `docs/issues/*.md`
- Agent prompt: `docs/PROMPT.md`

## Product Direction

**Polet AI Confidential DCA Smart Wallet**

Users deposit funds into a Polet smart wallet PDA, configure confidential numeric policy rules, grant an AI agent a temporary session key, and let the agent run a Jupiter-powered USDC -> SOL DCA strategy.

Core demo:

- DCA run: 5 USDC -> SOL
- Confidential max per run: 10 USDC
- Confidential daily cap: 20 USDC
- Block scenario: 25 USDC
- Allow scenario: 5 USDC

Track alignment:

- Encrypt/Ika: confidential numeric policy enforcement is core.
- Jupiter: Tokens + Price + Recurring compatibility + Swap V2 `/build` fallback.
- National Campus Hackathon Indonesia: Consumer App / DeFi safety for Indonesian users.

## Notes

- The final product uses one contract. Do not create a separate `polet_v2` program.
- Encrypt pre-alpha must be described honestly. Do not claim production confidentiality.
- Jupiter `/build` is preferred for smart wallet execution because Polet needs raw instruction control.
- Jupiter Recurring is primary only if compatible with the smart wallet flow; otherwise use Polet scheduler + Swap V2 `/build`.
- Encrypted allowlist/blocklist membership is out of scope for the first milestone.
